# Phase F — N=7 multi-resolution ensemble (img320 added)

This file documents adding a 7th model trained at image_size=320 (vs the existing six at 224) and the multi-resolution ensembling that lets it combine cleanly with the rest. It is the next step after `phase_e_n6_final_20260430.md`.

## What was added

1. **img320 model**: same `robustaug+EMA` recipe as the other six but with `image_size=320` and `batch_size=20` (memory headroom on MPS). Config: `configs/protocol_youtube_dataset_plus_local_videoonly_clean_robustaug_ema_img320.yaml`. Best epoch 10, EMA val_f1 = **0.7953** — the strongest single model so far on val (vs 224 best 0.7634).
2. **Multi-resolution ensemble script**: `scripts/eval_n_model_multires_ensemble.py`. Each config gets its own dataset / transform / dataloader so models with different `image_size` can co-exist; with `shuffle=False` and identical split filtering the per-frame ordering is verified to match across all configs (raises `[ERROR]` on mismatch).
3. **Aggregation paths supported**: mean, geomean, median, trimmed_mean (same set as the 224-only `eval_n_model_aggregations.py`).

## Cost vs benefit

- **Inference cost goes up**: 7 models × 2 (hflip TTA) = 14 forward passes per video, with one of those at 320×320 (≈2× FLOPs of a 224×224 forward). Roughly equivalent to 16 single-model 224 inferences. Still a few seconds per video on MPS, fine for non-realtime detection.
- **Training cost added**: 1 new model at 320×320 took ~3.5 hours on MPS (vs ~3 hours for a 224 model). One-shot cost.
- **Benefit**: AUC +3.5% rel, TPR@FPR=0.1% +197% rel (≈3×), with strong OOD generalization (AUC 0.992 / 0.991 on never-seen Face2Face / Deepfakes).

## Three recommended operating modes (final-final)

| mode | recipe | best for |
|---|---|---|
| **High-coverage** | N=7 median, no calibration | best F1, AUC, TPR@FPR=0.1% on test |
| **Strict-low-FPR alt** | N=6 median (no img320) | best TPR@FPR=1% on test (0.3130) |
| **Strict-calibrated** | N=6 geomean + per-model T-scaling | best ECE (0.0392) when calibrated probabilities matter |

The first mode is the new default; modes 2-3 are kept because they happen to win their narrow metrics. In production the N=7 ensemble + median aggregation + TTA hflip is the headline recommendation.
